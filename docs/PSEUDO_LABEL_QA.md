# Анализ качества псевдоразметки Teacher-модели

## Контекст

На текущем этапе проекта проводится генерация псевдоразметки пешеходов с помощью Teacher-модели pose estimation.

На вход Teacher-модели подаются кропы пешеходов, полученные по bounding box. На выходе ожидаются 17 ключевых точек и confidence score для каждого объекта.

Цель анализа - оценить качество псевдоразметки и определить стратегию фильтрации данных для обучения Student-модели.

## Группы примеров

### 1. Высококачественная разметка

Характеристики:

- confidence примерно `0.75-0.90+`;
- анатомически корректное расположение ключевых точек;
- хорошо определяются голова, плечи, локти, таз, колени и стопы;
- скелет соответствует реальной позе человека.

Такие примеры составляют основу будущего датасета и должны использоваться без дополнительных ограничений.

### 2. Частично корректная разметка

Характеристики:

- confidence примерно `0.50-0.75`;
- присутствуют окклюзии;
- часть конечностей скрыта объектами сцены;
- возможны небольшие ошибки в расположении отдельных точек.

Такие примеры важны для обучения, поскольку отражают реальные условия автономного транспорта:

- пешеходы за автомобилями;
- пешеходы за столбами;
- частичное перекрытие людьми друг друга;
- дальние объекты.

Несмотря на шум, подобные данные полезны для повышения устойчивости модели.

### 3. Низкокачественная разметка

Характеристики:

- confidence ниже `0.5`;
- заметные ошибки Teacher-модели;
- нарушение структуры скелета;
- отдельные точки располагаются вне тела человека;
- сильные ошибки на дальних и плохо видимых объектах.

Подобные примеры могут негативно влиять на обучение Student-модели.

## Основная проблема

Нельзя автоматически удалять все сложные примеры:

- дальних пешеходов;
- частично скрытых пешеходов;
- людей за автомобилями;
- людей за препятствиями.

Полный отказ от таких примеров сместит распределение данных. Student-модель будет обучаться только на идеальных изображениях и может плохо работать в реальных дорожных сценах.

## Стратегия baseline

Для первого baseline используется умеренная фильтрация.

Сохранять:

- примеры с mean confidence `> 0.5`;
- частично скрытых пешеходов;
- умеренно дальние объекты;
- случаи частичной окклюзии.

Удалять:

- экстремально маленькие bounding box;
- случаи с очень низкой уверенностью Teacher;
- явно сломанные скелеты;
- объекты, для которых предсказано слишком мало уверенных ключевых точек.

## Использование confidence

Confidence Teacher-модели нужно сохранять и использовать во время обучения Student.

Идея:

- качественные псевдоразметки дают больший вклад в функцию потерь;
- менее надежные псевдоразметки дают меньший вклад;
- сложные, но полезные дорожные случаи не удаляются полностью.

Такой подход позволяет использовать больше данных без сильного увеличения уровня шума.

## Предварительный вывод

На этапе первого baseline нецелесообразно стремиться к полностью очищенному датасету.

Рекомендуемый порядок:

1. Сгенерировать полный набор псевдоразметки.
2. Отфильтровать только очевидный мусор.
3. Сохранить confidence Teacher-модели.
4. Обучить первую Student-модель.
5. После первых метрик провести дополнительный анализ ошибок и скорректировать фильтрацию.

Это позволяет быстрее получить рабочий baseline и понять реальные ограничения данных и модели до начала сложной ручной очистки.
